Project Overview

For this assignment, I worked within a simple “dream job” scenario. The idea was to imagine being hired as a new software engineer at a tech company in Central Florida. My task was to help develop a real-time system — a type of system where timing is just as important as getting the correct result.

To make the scenario feel more realistic, it was placed in the healthcare technology world. Devices like patient monitors, insulin pumps, and even pacemakers all rely on software that must react quickly and reliably. If a heart rate sensor detects something dangerous, for example, the system needs to respond within seconds, not minutes. Real-time systems like these help ensure accuracy, safety, and quick reactions when it matters most.

This project gave me hands-on experience building a small version of a real-time application, practicing how timing, responsiveness, and predictable behavior all come together in embedded systems.
